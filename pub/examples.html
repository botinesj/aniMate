What is aniMate?
aniMate is a learning utility library that allows users to easily create animations representing plays or events in sports in a specified HTML Div element of a webpage. We refer to this HTML Div element as the animation field.

The developper dictates what exactly this HTML Div element is, what type of elements can be animated within it and what happens when the animations stop running.

Users are able to add elements selected by the developer anywhere within the designated animation field. These added elements can be assigned custom animations which can then be run simultaneously.

In general, the library is intended to be used for the creation of visualizations that are relevant to sports which involve a ball. Specifically, these visual animations are meant to be used as a teaching tool for coaches.
Features
  • The animations of all elements occur simultaneously.
  • The animation path of each element is marked by an outline.
  • The colour of the animation path's outline can be specified.
  • Animations can be paused and resumed.
  • There is a slider which can be dragged back and forth to temporarily move all elements along their animation paths.
  • The speed of each animation can be specified.
  • Multiple teams and players can be represented.
  • When an element has been added to the animation field, it can be moved within the animation field (but cannot leave the animation field) by dragging the element with the user’s mouse.
  • If an element with an animation path is dragged by the user, the animation path will automatically be updated to the element's new position. However, while dragging such an element, if any part of the updated animation path would result in the element leaving the animation field, it is removed from the element’s animation path.
  • Depending on the developpers choice, when all the animations have finished, the elements will either return to their starting positions with the exact same animation paths or the elements will remain in the final position of their animation path and their previous animation paths will be erased.
Example 1: Animations Reset
We create an animation field that allows users to build a football play. We want the user to be able to repeat and refine the play, that is, we want the animations to reset. Thus, we need to call setReset(true), along with the other methods of the library. The following code occurs in the HTML file:
<button id="footballButton"> Make Football Field the Animation Field </button>
<div id="animationField"></div>
The following code occurs in the CSS file:
#animationField {
...
background-image: url("americanFootbalField.jpeg");
background-size: 100% 100%;
position: relative;
...
}
The following code occurs in the JavaScript file. Please note that the order of the aniMate method calls must be the same as below for library to work:
const aMlibrary = new aM()

const player = document.createElement('div')
player.style = 'width: 40px; height: 40px; border-radius: 50%; background-color: Blue;'

const ball = document.createElement('div')
ball.style = 'width: 30px; height: 30px; border-radius: 50%; background-color: Brown;'

const hInteracter = aMlibrary.horizontalInteracter()

const footballButton = document.querySelector('#footballButton')
footballButton.addEventListener('click', footballInitiate)

function footballInitiate(e) {
// animationField is the id of the Div element containing the football field below
aMlibrary.initiate('#animationField')
aMlibrary.setReset(true)
$('#animationField').after( hInteracter);
aMlibrary.setSlider();
aMlibrary.setPlayer(player)
aMlibrary.setBall(ball)
}
Now that everything has been set up, click the "Make Football Field the Animation Field" button directly below. Then, click anywhere within the football field, and follow the instructions that pop up. Note the horizontal interacter HTML element that appears below the football field. Observe that after hitting the "Play Animations" button, when all the animations finish, they return to their starting positions with the exact same animation paths.
Example 2: Animations Do Not Reset
We create an animation field that allows users to demonstrate certain soccer scenarios and how to continue after they occur. That is, we want the animations to not reset. Thus, we need to call setReset(false) instead of setReset(true). The following code occurs in the HTML file:
<button id="soccerButton"> Make Soccer Field the Animation Field </button>
<div id="animationField2"></div>
The following code occurs in the CSS file:
#animationField2 {
...
background-image: url("soccerField.png");
background-size: 100% 100%;
position: relative;
...
}
The following code occurs in the JavaScript file. Please note that the order of the aniMate method calls must be the same as below for the library to work:
const soccerButton = document.querySelector('#soccerButton')
soccerButton.addEventListener('click', soccerInitiate)

function soccerInitiate(e) {
// animationField2 is the id of the Div element containing the soccer field below
aMlibrary.initiate('#animationField2')
aMlibrary.setReset(false)
$('#animationField2').after( hInteracter);
aMlibrary.setSlider();
aMlibrary.setPlayer(player)
aMlibrary.setBall(ball)
}
Now that everything has been set up, click the "Make Soccer Field the Animation Field" button directly below. Then, click anywhere within the soccer field, and follow the instructions that pop up. Note the horizontal interacter HTML element that appears below the soccer field. Observer that after hitting the "Play Animations" button, when all the animations finish, they do not return to their starting positions and the completed animation paths are deleted.
